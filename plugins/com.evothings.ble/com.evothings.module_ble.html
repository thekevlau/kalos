(static) startScan(win, fail)

Starts scanning for devices.

Found devices and errors will be reported to the supplied callbacks.

Will keep scanning indefinitely until you call stopScan().

To conserve energy, call stopScan() as soon as you've found the device you're looking for.

Calling this function while scanning is in progress has no effect?

AdvertisementData

Information extracted from a scanRecord. Some or all of the fields may be undefined. This varies between BLE devices. Depending on OS version and BLE device, additional fields, not documented here, may be present.
Type:
  • Object
Properties:
Name Type Description
kCBAdvDataLocalName string The device's name. Equal to DeviceInfo.name.
kCBAdvDataChannel number A positive integer, the BLE channel on which the device listens for connections. Ignore this number.
kCBAdvDataIsConnectable boolean True if the device accepts connections. False if it doesn't.
kCBAdvDataServiceUUIDs array Array of strings, the UUIDs of services advertised by the device. Formatted according to RFC 4122, all lowercase.
kCBAdvDataManufacturerData string Base-64-encoded binary data. This field is used by BLE devices to advertise custom data that don't fit into any of the other fields.
